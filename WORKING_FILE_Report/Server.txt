Node.js WebSocket Server & DB (server.js) - Working File Report

1. Architecture and Setup:
- Built with Express.js to serve static dashboard files and a real-time WebSocket Server (`ws`) to handle two-way communication between the ESP32 and web users.
- Connects to a MongoDB database (`motor_data6c`) via Mongoose to permanently store motor activity logs.

2. Client Identification & Routing:
- The server acts as a middleman (Bridge).
- When a new WebSocket client connects, the server waits 2000ms to identify it. If the client sends `{"type":"esp32-identify"}`, it is marked as the "Hardware Client" (esp32Client).
- If it doesn't send that payload, it is marked as a "Web Browser Client" (webClients). 
- This enables it to broadcast ESP32's JSON status to ALL connected web users in real-time, and forward commands from any web user explicitly down to the targeted ESP32.

3. Motor Analytics & Logging Engine:
- The server intelligently tracks the `motorStatus` field coming from the ESP32.
- When it sees the motor transition from "OFF" to "ON", it captures the `motorStartTime`.
- When the motor transitions from "ON" to "OFF", it captures the stop time, calculates the running duration, and checks if it ran for more than 2 seconds (to ignore bounce/noise).
- If it is a valid run, it converts the timestamps to Bangladesh Standard Time (BST) and saves a structured document into the MongoDB `MotorLog46` collection.

4. Log API & Queries:
- Responds to the "GET_LOG_PAGE" command from the web dashboard.
- Uses Mongoose `skip()` and `limit()` to paginate records (10 logs per page) rather than loading everything at once.
- Supports Date-range filtering (startDate to endDate) by querying the MongoDB `startTime` field intelligently.
- Can process the "CLEAR_LOGS" command to drop all records from the database and instantly broadcast the "Logs Cleared" update to all connected users.
