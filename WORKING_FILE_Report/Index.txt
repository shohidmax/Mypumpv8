Dashboard Single-Page App (index.html) - Working File Report

1. UI Frameworks & Styling:
- Uses TailwindCSS for utility-first layout building and DaisyUI for beautiful, pre-built components (Cards, Buttons, Modals, Tooltips).
- Fully responsive across desktop and mobile viewing with light and dark mode toggling stored in `localStorage`.
- Real-time animated gear icon dynamically starts spinning when the motor is detected as "ON".

2. Real-time WebSocket Connectivity:
- Opens a direct WebSocket connection to `wss://pumpv6c3.espserver.site`.
- Automatically tracks connection status. If the internet drops, it grays out UI buttons and displays a "Disconnected" badge, attempting to reconnect every 3 seconds silently.
- Receives `statusUpdate` messages from the server and instantly paints the DOM: updates text from "ON"/"OFF", dynamically changes text colors (Success Green vs Info Blue), updates the WiFi signal bars, and sets the IP address in a tooltip.

3. Motor Control System:
- "Start Motor" and "Stop Motor" buttons open a DaisyUI Dialog (Modal) to ask for user confirmation to prevent accidental triggering.
- Once confirmed, sends `{"type": "command", "command": "RELAY_1"}` or `RELAY_2` to the WebSocket Server.
- Provides pop-up toast notifications across the bottom of the screen indicating action status.

4. Log Management & Data Visualization:
- Includes a sophisticated Table wrapper for viewing "Motor Activity Logs".
- Sends paginated payload requests and paints rows dynamically directly from the database.
- Client-side PDF generation: Integrates `jsPDF` and `jspdf-autotable` to export the currently loaded local logs strictly into a downloaded PDF file named `motor_log_YYYY-MM-DD.pdf`.
- Filters via date inputs dynamically bound to the server side query.
